2. What gets paywalled

Free users can:
	•	upload or connect a project
	•	see a partial analyzer report
	•	view top-level blockers
	•	preview readiness score
	•	create one sandbox project

Paid users can:
	•	unlock full analyzer results
	•	use the guided fix flow
	•	generate packaging/wrapper output
	•	connect GitHub / Codemagic / App Store Connect
	•	save multiple projects
	•	run pre-submission validation
	•	submit through POSTAPP workflows

Per-app submission fee triggers when user:
	•	starts final submission orchestration
	•	generates final submission package
	•	triggers production build handoff
	•	uses submission/resubmission workflow

⸻

3. Product logic

Recommended gating rule

Do not charge too early.

Best sequence:
	1.	User uploads app
	2.	POSTAPP analyzes it
	3.	User sees value
	4.	User upgrades to continue
	5.	User pays per-app fee only when moving into final submission pipeline

That way the system feels fair.
